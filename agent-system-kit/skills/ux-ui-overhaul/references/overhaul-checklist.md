# Overhaul Checklist

## Discovery

- [ ] Capture all routes/views in scope.
- [ ] Capture all required content blocks per view.
- [ ] Identify critical interactions (search, filters, editing, save/publish, navigation).

## Personalization

- [ ] Translate user background into visual principles.
- [ ] Document visual anti-patterns to avoid.
- [ ] Define 1-2 naming concepts for the design direction.

## Design System

- [ ] Palette includes semantic usage guidance (not just hex values).
- [ ] Typography includes display + body roles.
- [ ] Motion guidance includes when to animate and when not to.
- [ ] Component style rules are reusable across pages.

## Mockups

- [ ] Include desktop and mobile states.
- [ ] Include all major app page types.
- [ ] Clearly label frames and intended interactions.

## Compatibility

- [ ] Confirm no content model changes required.
- [ ] Confirm no backend contract changes required.
- [ ] List optional enhancements separately from required parity.

## Approval Gate

- [ ] Provide explicit approval checklist.
- [ ] Separate "approved" vs "needs revision" feedback categories.
