# Figma Brief Template

## Objective

Redesign the existing product experience for stronger visual identity and usability while preserving behavior and content structure.

## Constraints

- Keep routes and feature behavior unchanged.
- Keep content model/schema unchanged.
- Focus on UI/UX organization and visual language.

## Required Frames

1. Home / Landing
2. Work / Experience
3. Projects
4. Blog List
5. Blog Detail
6. Authoring Dashboard (if applicable)
7. Authoring Editor (if applicable)
8. Mobile key screens

## Component Inventory

- Navigation (desktop + mobile)
- Section containers
- Cards
- Chips/tags
- Status badges
- Buttons and action rows
- Form controls

## Tokens

- Brand colors (primary/secondary/accent/status)
- Typography scale (display/heading/body/caption)
- Spacing system
- Radius/shadows

## QA Criteria

- All existing functionality represented in new UI.
- Readability and accessibility pass.
- Clear hierarchy and reduced cognitive load.
